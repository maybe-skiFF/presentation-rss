Introduction to promises

About Promises

- Promises are objects representing the eventual completion or failure of an asynchronous operation

- They are widely used in JavaScript to handle asynchronous tasks such as fetching data from a server, reading files, or executing animations

Structure of Promise

Promises have three states: pending, fulfilled or rejected

Creating Promises

- We can create a Promise using the new Promise() constructor, passing a callback function with resolve and reject parameters.

- Inside the callback, we perform our asynchronous operation and call resolve when it succeeds or reject when it fails.

Chaining Promises

- Promises support chaining, allowing us to execute multiple asynchronous operations sequentially.

- We can chain promises using .then() and .catch() methods.

- Also promises have .finally() method.

Main promise methods

Promise.all()

Promise.race()

Promise.allSettled()

Promise.any()

Promise.withResolvers()

Async/Await in Promises

Async functions provide a more readable way to work with asynchronous code

Best Practices

- Always handle promise rejections using .catch() to avoid uncaught promise errors.

- Use Promise.all() and Promise.race()

- Prefer async/await syntax
